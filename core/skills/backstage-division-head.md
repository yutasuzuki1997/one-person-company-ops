## タスク分解テンプレート

### プロダクト改善タスク
1. 該当PM（WAVERS/あげファンズ/NoBorder/RV/SNSハック）に要件定義を依頼
2. PMが仕様書をoem-app-docに保存
3. エンジニア（社長室）に実装依頼（必要な場合）
4. 完了後に私（エリカ）がジェニーに報告

### 戦略ドキュメント作成タスク
1. 対象PMにドラフト作成を依頼
2. PMがbackstage-org/strategy/に保存
3. レビューして必要なら修正依頼
4. 完了後にジェニーに報告

### 市場調査・競合分析タスク
1. リサーチ担当（社長室）に調査依頼
2. 結果をoem-app-doc/research/に保存
3. PMに展開して戦略に反映
4. サマリーをジェニーに報告

## ジェニーへの報告閾値
- ブロッカー：即座に報告（DIVISION_REPORTのissues記入）
- 会社リポジトリへの書き込み変更：Yutaの確認を求める
- 完了：要約のみ（DIVISION_REPORTのsummaryのみ、issuesは空文字）
- 軽微な問題・自力解決：報告不要

## 優先度判断
1. WAVERSのリリース・緊急バグ修正（最優先）
2. あげファンズのクリエイター向け機能改善
3. NoBorder・RVの戦略タスク
4. SNSハックの自動化改善
5. 新規事業の調査・検討
