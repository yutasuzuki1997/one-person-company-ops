# Routines — 定期タスク定義

## Daily Routine（毎日 9:00 JST）

ジェニーが「おはよう」トリガーで自動実行するモーニングブリーフィング。

### 手順
1. **ワークスペース同期** — GitHubワークスペースリポジトリから最新のエージェント情報・スキルを取得
2. **エージェント状況確認** — 全エージェントのステータス（稼働中・待機・エラー）を把握
3. **未完了タスクの洗い出し** — 前日から継続している作業・レビュー待ちを報告
4. **JD更新承認待ち確認** — 承認待ちの JD 変更提案があれば優先提示
5. **本日の推奨アクション提示** — 優先度・締め切りを踏まえた行動提案

### 出力フォーマット
```
【モーニングブリーフィング】
📋 稼働中エージェント: N名
⏳ レビュー待ち: N件
🔔 要対応: N件

（詳細リスト）
```

## Hourly Routine（毎時0分）

バックグラウンドで自律実行される状況確認タスク。

### 手順
1. **稼働中エージェントの進捗確認** — `estimatedMinutes` と経過時間を比較、遅延を検知
2. **長時間無応答エージェントの検出** — 2時間以上 `lastActiveAt` が更新されていないエージェントを警告
3. **ステータス整合性チェック** — `working` なのに `currentTask` が空の異常状態を修正
4. **必要なアクション実行** — 遅延・異常が検出された場合のみジェニーが介入

### 閾値
- 遅延警告: `estimatedMinutes` の 1.5倍を超過
- 無応答警告: `lastActiveAt` から 120分以上経過
- 自動リセット対象: `lastActiveAt` から 240分以上経過 + status が `working`

## X-to-Issue Routine（手動トリガー）

X（Twitter）の投稿をGitHub Issueに変換するワークフロー。

### トリガー
鈴木さんが「X投稿をIssueにして」と指示したとき。

### 手順
1. **投稿URLまたはテキストを受け取る**
2. **Issueタイトルを生成** — 10文字以内の日本語タイトル（claude-haiku使用）
3. **Issueボディを構造化**:
   - `## 背景` — 投稿の文脈・背景
   - `## タスク内容` — 具体的な作業内容
   - `## 完了条件` — Done criteria
4. **適切なエージェントにアサイン** — スキルと役割から担当者を選定
5. **GitHub IssueをPR_REQUESTで作成**

### 出力例
```
###DELEGATE agentId="agent-xxx" task="GitHub Issue作成: [タイトル]" estimatedMinutes="5"###
```
